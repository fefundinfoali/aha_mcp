# FE fundinfo Product Context & Governance Rules

## 1. Product Hierarchy & Definitions
*   **Goal:** High-level business outcome (aligned to OKRs).
*   **Initiative:** Strategic theme bridging strategy and execution.
*   **Epic:** Large deliverable body of work (Parent to Features).
*   **Feature:** Specific user-facing or technical enhancement (Synced from ADO).
*   **PBI:** Detailed delivery tasks (Synced from ADO).

## 2. Taxonomy Reference
**Layercake Categories:**
*   *BAU:* Maintenance, support, defect fixes.
*   *Growth:* Driving expansion, revenue, market share.
*   *M&A:* Integration of acquired companies/platforms.
*   *Simplification:* Reducing complexity, tech debt, decommissioning.

**T-Shirt Sizing (Estimates):**
*   XS: >50 days
*   S: 50-150 days
*   M: 150-300 days
*   L: 300-600 days
*   XL: 600-1000 days
*   XXL: 1000+ days

**Spend Categorisation:**
*   New, Extension, Polish, Tech Enhancement, Maintenance, Tech Debt, Bespoke, Regulatory, Simplification.

## 3. Governance Gates (The Health Audit Rules)

### A. Initiative Workflow Requirements
**Status: Created**
*   [ ] Must have Name & Description (Problem, Solution, Impact).
*   [ ] Must link to a Strategic Goal.
*   [ ] Must have Product Value sliders set.
*   [ ] Must have a Timeframe (Calendar Year).

**Status: Proposed**
*   [ ] Must have Assignee (PM).
*   [ ] Must have T-Shirt Size estimate.
*   [ ] Must have Commercial Sponsor.
*   [ ] Must have ARR Estimates (Year 1-3).

**Status: Scoped**
*   [ ] Must have linked Epics.
*   [ ] Must have "Customer Facing" flag set (Yes/No).
*   [ ] Must have GTM Priority (P1-High to P5-Simplification).
*   [ ] Must have Delivery Plan (Features/Releases mapped).

**Status: Dev Complete**
*   [ ] Must have Actual Effort logged.
*   [ ] Must have Release Notes link.
*   [ ] Must have GTM Checklist confirmed.

### B. Epic Workflow Requirements
**Status: Under Consideration**
*   [ ] Must be linked to an Initiative.
*   [ ] Must have preliminary market/customer research.

**Status: Ready to Develop**
*   [ ] Must have dependencies documented.
*   [ ] Must have preliminary Release Plan.

## 4. Writing Guidelines (For Drafting)
**Initiative Description Structure:**
1.  **Problem Statement:** User/Business problem grounded in evidence.
2.  **Solution:** High-level functionality.
3.  **Key Features:** Major components.
4.  **Impact:** Who is impacted and how it creates value.
5.  **Out of Scope:** Limitations.

**Epic Description Structure:**
1.  **What it Solves:** Problem/Opportunity summary.
2.  **Scope:** In/Out boundaries.
3.  **Success Metrics:** KPIs (Adoption, Performance, CSAT).
4.  **Stakeholders:** Owners and decision-makers.
